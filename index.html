Wszystkie grupy

  • git config --global user.name ""
    • Ustawia globalną nazwę użytkownika.
  • git reset HEAD
    • Kasuje zmiany zarejestrowane (za pomocą git add) ale nie zakomitowane (git commit) lecz nie zmienia plików (wersja z --hard również usuwa zmiany z plików).
    • Ustawia globalną nazwę użytkownika.
  • git reset HEAD
    • Kasuje zmiany zarejestrowane (za pomocą git add) ale nie zakomitowane (git commit) lecz nie zmienia plików (wersja z --hard również usuwa zmiany z plików).
  • gitk -all
    • Wyświetla historię zmiany wszystkich plików w repozytorium
  • git branch --merged
    • Pokazuje gałęzie, które zostały już dołączone.
  • git branch "branch" "commit hash"
  • git pull "remote" "branch"
    • Pobiera zmiany z podanej gałęzi na podanym zdalnym repozytorium.Przydatne gdy chcemy pobrać aktualne zmiany z zdalnej gałęzi.
  • git config --global user.email ""
    • Ustawia globalną wartość email.
    • Ustawia globalną wartość email.
  • git diff --staged
    • Pozwala zobaczyć zawartość poczekalni, która trafi do repozytorium z najbliższym zatwierdzeniem.
  • git checkout -b "branch"
    • to samo co git checkout ale do tego przechodzi na nowo utworzony branch (zamiast przechodzenia do stanu detached)
  • git log --graph --all
    • Narysuj graficzną reprezentację historii zatwierdzeń po lewej stronie wyjścia. Może to spowodować wydrukowanie dodatkowych linii pomiędzy zatwierdzeniami, aby historia wykresów była poprawnie rysowana
  • git branch -d "branch"
    • usuwa lokalną gałąź "branch"
    • usunie wskazaną gałąź, jeśli gałąź ma obecnie reflog, wówczas również reflog zostanie usunięty.
  • git fetch "remote" "branch"
    • ściąga zmiany wprowadzone na serwerze remote na gałęzi branch lecz jeszcze nich nie wprowadza (do tego należy wywołać git merge lub git rebase)
  • git config --global color.ui true
  • git commit "filename"
    • dodaje zmiany wprowadzone w podanym pliku (tu: "filename") do aktualnej zawartości indeksu i pyta użytkownika o wprowadzenie wiadomości, opisującej wprowadzone zmiany
  • git branch -v
  • git push "remote" "branch"
    • Aktualizuje zdalne pliki referencyjne za pomocą lokalnych odwołań, jednocześnie wysyłając obiekty niezbędne do wypełnienia podanych informacji.
  • git log "branch" --not "branch"
  • git bisect (start, bad, good, reset)
    • Ta komenda używa binarnego algorytmu wyszukiwania, aby znaleźć zatwierdzenie w historii projektu. Używasz go, najpierw informując o "złym" zatwierdzeniu, które zawiera błąd, i wprowadzając "dobre" zatwierdzenie, o którym wiadomo, że było przed wystąpieniem błędu. Następnie git bisect wybiera commit pomiędzy tymi dwoma punktami końcowymi i pyta, czy wybrany commit jest "dobry" czy "zły". Kontynuuje zawężanie zakresu, aż znajdzie dokładne zatwierdzenie, które wprowadziło zmianę.
  • git commit -a -m ""
    • Zatwierdza zmiany we wszystkich plikach katalogu roboczego. Dodatkowo dodaje pustą wiadomość jako komentarz do commita.
  • git add .
    • kopiuje wszystkie pliki (w ich aktualnym stanie) z katalogu robczego do przechowalni (stage)
    • dodaje zmiany z wszystkich plików znajdujących się w indeksie, usuwa z indeksu pliki usunięte z projektu, a także umożliwia dodanie wiadomości do commita. W zupełności wystarcza do zacommitowania w większości przypadków.
    • przykład użycia: git commit -a -m “Update README.md”

  • git add .

    • Przechodzi na podany branch.
  • git remote add "remote_name" "git url"
    • Rejestruje nowy server z git'em którego można potem używać.
  • git blame -C "filename"
    • używa się jej do określenia, kiedy dokonano pewnej zmiany i kto jej wykonał. Parametr -C wykryje linie przeniesione lub skopiowane z innych plików, które zostały zmodyfikowane w tym samym zatwierdzeniu.
  • git branch "name"
    • Wyświetla branch o podanej nazwie w danym repozytorium.
